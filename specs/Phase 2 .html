Strategic Overview

Executive Mandate

"Project Sovereign Algorithm" represents a tectonic shift in the grand strategy genre. It is a persistent world simulation that treats digital infrastructure as contested territory, replacing reflexive high-APM gameplay with long-form strategic planning and machine learning orchestration.

Primary Objective

Provide a platform for 1M+ global participants to compete in a zero-latency world-state where the "Fog of War" is generated by cryptographic asymmetry.

Technical Philosophy

Cloud-native, event-driven architecture that abstracts the complexities of AI training into intuitive gameplay loops.

Core Loop

The Epoch Cycle

Sovereign Algorithm does not run in "real-time" ticks. It runs in **Epochs**—15-minute global windows where strategy is planned, then executed simultaneously by the cloud engine.

1

Planning Phase (00:00 - 10:00)

Human directors issue orders, negotiate treaties via Gemini-LLM, and allocate Compute Units (CU).

2

Sim Phase (10:00 - 14:00)

The world state is locked. GKE clusters execute 1M+ Sentinel agent simulations based on RL policies.

3

Transition (14:00 - 15:00)

Firestore batch updates synchronize the results globally. News alerts are generated by LLMs.

Machine Learning

Sentinel RL Training

Players do not micromanage units. They act as "Neural Architects," tuning Reinforcement Learning reward functions to govern autonomous agent behavior.

The Reward Function Policy

Every Sentinel uses a **Proximal Policy Optimization (PPO)** model trained on Vertex AI. The player modifies the following weights:

  • ● **Persistence:** Value of maintaining a foothold vs retreating.
  • ● **Stealth:** Penalty multiplier for being detected by rival AI.
  • ● **Efficiency:** Reward for minimizing CU consumption per breach.
